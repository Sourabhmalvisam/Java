✅ Java Primitive Data Types (Interview-Ready Notes)

🔷 1. What is a Variable?

  • A variable is a container to store data.
  • Declared using:
datatype variableName = value;
  • Java is:
    • Statically typed – Type must be declared.
    • Strongly typed – No implicit type conversion if it causes loss of precision.

🔷 2. Variable Naming Rules

  • Case-sensitive
  • Can include letters, digits, $, _ (not start with digits)
  • Must not use Java reserved keywords (e.g., class, int)
  • Constants: ALL_CAPS (e.g., PI, MAX_SIZE)
  • Variables: camelCase (e.g., employeeId)

🔷 5. Type Conversion

🔸 Widening (Implicit)

  • Small to large type (safe)
int a = 10;
long b = a; // auto-converted

🔸 Narrowing (Explicit/Manual)

  • Large to small type (data loss possible)
int x = 130;
byte y = (byte) x; // y becomes -126

🔸 Promotion in Expressions

  • Smaller types (byte, short) are promoted to int in expressions.
byte a = 100, b = 27;
byte sum = (byte) (a + b); // must cast explicitly

🧠 Interview Tip: Always expect questions on implicit vs explicit type casting and data loss scenarios.

🔷 7. Floating Point Representation (float/double)

  • Stored using IEEE 754 standard:
    • Float: 1 sign bit, 8 exponent bits, 23 mantissa bits
    • Double: 1 sign bit, 11 exponent bits, 52 mantissa bits

🧠 Why float != precise?

  • Decimal values like 0.7f are not exactly representable, due to binary conversion. It leads to precision errors.
  • Use BigDecimal for financial calculations.

🔷 8. Binary Representation and Overflow

  • Two's Complement is used for signed values.
  • In case of overflow:
byte b = (byte) 130;  // becomes -126

🧠 Interview Ask: Explain how signed overflow works and how Java handles it.
