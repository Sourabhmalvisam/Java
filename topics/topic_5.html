How FLOAT and DOUBLE Number Stored in Memory? | IEEE 754

Java uses IEEE 754 standard for floating-point number representation — same as most modern languages (C, C++, Python, etc.)

🧠 Quick Summary:

TypeSize (bits)PrecisionIEEE Format
float32-bit~7 digitsSingle Precision
double64-bit~15 digitsDouble Precision

🧬 Internal Binary Structure (IEEE 754)

1️⃣ Float (float - 32-bit total)

| 1 bit sign | 8 bits exponent | 23 bits mantissa |

2️⃣ Double (double - 64-bit total)

| 1 bit sign | 11 bits exponent | 52 bits mantissa |

✅ What does each part do?

PartRole
Sign Bit0 = positive, 1 = negative
ExponentDetermines the magnitude or range (like scientific notation's power of 10)
Mantissa / FractionStores the significant digits (precision)

🔍 Example: How is 5.75 stored as float?

➤ Step-by-step:

  • Convert 5.75 to binary:
    5 = 101
0.75 = 0.11 (because 0.5 + 0.25)
So 5.75 = 101.11
  • Normalize binary to scientific form:
    1.0111 × 2^2
  • IEEE format:
    • Sign bit = 0 (positive)
    • Exponent = 127 (bias) + 2 = 129 → 10000001
    • Mantissa = 01110000000000000000000

👉 Final 32-bit float:

0 10000001 01110000000000000000000

🔐 Bias in Exponent

  • Bias is used to represent both negative and positive exponents
    • For float → bias = 127
    • For double → bias = 1023

Example:

If exponent = 5

Then stored value = 127 + 5 = 132 in float

📌 Java Examples to See Precision Differences

public class FloatDoubleDemo {
    public static void main(String[] args) {
        float f = 1.0f / 3.0f;
        double d = 1.0 / 3.0;

        System.out.println("Float:  " + f);  // Float:  0.33333334
        System.out.println("Double: " + d);  // Double: 0.3333333333333333
    }
}

⚠️ Common Interview Mistakes to Avoid

MythReality
float is more accurate❌ It has less precision than double
You can compare floats directly❌ Floating point comparison needs tolerance
float/double are always exact❌ They lose precision on big/small numbers

🧪 Interview Tip: Use EPSILON for Comparisons

double a = 0.1 + 0.2;
double b = 0.3;
if (Math.abs(a - b) < 1e-9) {
    System.out.println("Equal");  // ✅ Correct way
}
