Java Memory Management and Garbage Collection

✅ Java Memory Management

Java uses automatic memory management handled by the JVM using Garbage Collector (GC).

🔷 Types of Memory in Java

✅ 1. Stack Memory

  • Stores:
    • Primitive data types
    • References to objects in heap
    • Method call frames
  • Each thread has its own stack.
  • LIFO order (Last-In-First-Out)
  • When full: java.lang.StackOverflowError

🧠 Scope-based deletion: Variables are removed when they go out of scope.

✅ 2. Heap Memory

  • Stores:
    • Objects
    • Class instances
    • Arrays
  • Shared among all threads.
  • Managed by Garbage Collector (GC)
  • Organized into Young Gen, Old Gen, and Metaspace.

🔷 Heap Memory Breakdown

🧠 Memory Areas:

SectionPurpose
Young GenerationNewly created objects, includes Eden + Survivor (S0/S1)
Old GenerationLong-living, aged objects
MetaspaceClass metadata, constants, static data (post-Java 8)

🔷 Object Lifecycle (Garbage Collection)

  1. New object → Eden
  1. GC runs → Moves surviving objects to S0
  1. On next GC → from S0 to S1 and vice versa
  1. If object survives multiple GC cycles (age threshold) → moves to Old Generation

🔷 Reference Types

TypeBehavior
StrongDefault. GC never deletes until reference removed
WeakGC may delete even if referenced
SoftDeleted only when memory low
PhantomUsed for cleanup post-finalization (not in PDF, but often asked)

✅ Manual GC Trigger (Not Guaranteed)

System.gc(); // Hint to JVM, not guaranteed

🔷 GC Algorithms

GC TypeDescription
Serial GCSingle-threaded GC (slow, used in client devices)
Parallel GCMulti-threaded, pauses application threads during GC (used in Java 8)
CMS (Concurrent Mark & Sweep)Concurrent GC, app threads mostly continue running
G1 GCReplaces CMS, handles both young + old generation, supports compaction and minimal pause time
ZGC/Shenandoah (latest)Ultra-low pause GC for large heaps

🔷 Mark & Sweep

  • Mark phase: Mark live (reachable) objects.
  • Sweep phase: Remove unmarked (unreachable) objects.
  • Compaction: Moves remaining objects to make free memory contiguous.

✅ Metaspace (Post Java 8)

  • Stores class definitions, static variables, constants.
  • Replaces PermGen (which had fixed size and was prone to OutOfMemoryError).
  • Metaspace is dynamically resizable.

🔷 Example Walkthrough (Interview Ready)

public class Person {
    int age = 10;
    String name = "John";
}

public static void main(String[] args) {
    Person p = new Person();
}

🧠 In memory:

StackHeapString Constant Pool
p → referencePerson object: age=10"John"
name → SCP["John"]

When main() ends:

  • Stack is cleared
  • GC deletes the Person object (if no other reference)
