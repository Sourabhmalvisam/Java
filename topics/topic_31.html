Lombok

✅ What is Lombok?

Project Lombok is a Java library that reduces boilerplate code using annotations like @Getter, @Setter, @Builder, etc.

It injects code into compiled .class files using annotation processing, so code looks minimal in source but complete after compilation.

💡 Why IntelliJ shows errors sometimes?

Lombok-generated methods aren't visible to the IDE unless:

  1. Lombok plugin is installed
  1. Annotation processing is enabled

Without this, your IDE will show red error squiggles even if the code compiles and runs fine.

⚙️ Required Setup

In pom.xml:

<dependency>
  <groupId>org.projectlombok</groupId>
  <artifactId>lombok</artifactId>
  <version>1.18.30</version> <!-- Use latest -->
  <scope>provided</scope>
</dependency>

IDE Setup (IntelliJ):

  • Install Lombok Plugin
  • Enable Annotation Processing in Preferences → Build Tools → Compiler → Annotation Processors

🔟 10 Most Frequently Used Lombok Features

1️⃣ val and var – Local Variable Inference

import lombok.val;
import lombok.var;

public class ValVarExample {
    public static void main(String[] args) {
        val name = "Ganesh"; // final String
        var age = 25;         // int

        // name = "Another"; ❌ Error (final)
        age = 26;              // ✅ Allowed
        System.out.println(name + ", age " + age);
    }
}
✅ Only works for local variables, not fields or method params.

2️⃣ @NonNull – Auto Null Check

import lombok.NonNull;

public class NonNullExample {
    public void setName(@NonNull String name) {
        System.out.println("Hello " + name);
    }

    public static void main(String[] args) {
        new NonNullExample().setName(null); // 💥 Throws NullPointerException
    }
}

3️⃣ @Getter and @Setter

import lombok.Getter;
import lombok.Setter;

public class GetterSetterExample {
    @Getter @Setter
    private String name;

    public static void main(String[] args) {
        GetterSetterExample obj = new GetterSetterExample();
        obj.setName("Ganesh");
        System.out.println(obj.getName());
    }
}
📌 Can also be used at class level.

4️⃣ Class-Level Getter/Setter

import lombok.Getter;
import lombok.Setter;
import lombok.AccessLevel;

@Getter
@Setter
public class User {
    private String name;
    private int age;

    @Setter(AccessLevel.NONE)
    private String role = "admin"; // ❌ No setter generated

    public static void main(String[] args) {
        User u = new User();
        u.setName("Ganesh");
        u.setAge(30);
        System.out.println(u.getName() + ", age: " + u.getAge());
    }
}

5️⃣ @ToString

import lombok.ToString;

@ToString
public class ToStringExample {
    private String name = "Ganesh";
    private int age = 30;

    public static void main(String[] args) {
        System.out.println(new ToStringExample()); // ToStringExample(name=Ganesh, age=30)
    }
}
Can exclude fields via @ToString(exclude = "fieldName")

6️⃣ @NoArgsConstructor, @AllArgsConstructor, @RequiredArgsConstructor

import lombok.*;

@RequiredArgsConstructor
@AllArgsConstructor
@NoArgsConstructor
public class ConstructorExample {
    @NonNull private String name;
    private int age;

    public static void main(String[] args) {
        ConstructorExample c1 = new ConstructorExample("Ganesh");
        ConstructorExample c2 = new ConstructorExample("Ganesh", 28);
        ConstructorExample c3 = new ConstructorExample();
    }
}

7️⃣ @EqualsAndHashCode

import lombok.EqualsAndHashCode;

@EqualsAndHashCode
public class EqualsHashExample {
    private String name = "Ganesh";
    private int id = 101;

    public static void main(String[] args) {
        EqualsHashExample e1 = new EqualsHashExample();
        EqualsHashExample e2 = new EqualsHashExample();
        System.out.println(e1.equals(e2)); // true
    }
}
Uses all non-static and non-transient fields by default.

8️⃣ @Data – All-in-One (DTO Booster)

import lombok.Data;

@Data
public class Employee {
    private final String name;
    private int salary;

    public static void main(String[] args) {
        Employee e = new Employee("Ganesh");
        e.setSalary(50000);
        System.out.println(e);
    }
}
Includes: @ToString, @EqualsAndHashCode, @Getter, @Setter, @RequiredArgsConstructor

9️⃣ @Value – Immutable Data Class

import lombok.Value;

@Value
public class ImmutablePerson {
    String name;
    int age;

    public static void main(String[] args) {
        ImmutablePerson p = new ImmutablePerson("Ganesh", 29);
        System.out.println(p); // ImmutablePerson(name=Ganesh, age=29)
    }
}
Immutable, final fields, no setters, class is final.

🔟 @Builder – Fluent Object Construction (Builder Pattern)

import lombok.Builder;

@Builder
public class Car {
    private String model;
    private int year;

    public static void main(String[] args) {
        Car c = Car.builder()
                   .model("Tesla")
                   .year(2024)
                   .build();
        System.out.println(c);
    }
}
Excellent for constructing complex objects.

🔁 @Cleanup – Auto-close Resources (Like try-with-resources)

import lombok.Cleanup;
import java.io.*;

public class CleanupExample {
    public static void main(String[] args) throws Exception {
        @Cleanup FileWriter writer = new FileWriter("test.txt");
        writer.write("Hello, Lombok!");
    }
}

🧠 Interview Insights

QuestionInsight
Why does Lombok need a plugin?IDEs don’t run annotation processors by default.
Is Lombok runtime or compile-time?Compile-time (injected into .class files).
Does it impact performance?No runtime impact, just cleaner code.
Can it be disabled easily?Yes, just remove dependency or disable processing.
