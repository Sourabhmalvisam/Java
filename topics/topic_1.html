OOPs Concept in Java

üî∑ OOPs Overview

  • OOP = Object-Oriented Programming
  • Real-world entities (e.g., Car, ATM, Bike) are treated as objects
  • Compared to procedural programming:
    • Procedural: Focus on functions, no data hiding
    • OOP: Focus on data, supports data hiding, overloading, inheritance, and reusability

üî∑ Objects & Classes

  • Object: Has Properties (State) and Behavior (Methods)
    • Example: Dog ‚Üí age, color (properties); bark, sleep (behaviors)
  • Class: Blueprint to create objects
    • Declared with class keyword
class Student {
    int age;
    String name;
    String address;

    void updateAddress() { ... }
    int getAge() { return age; }
}
Student engStu = new Student();

üî∑ 1st Pillar: Abstraction

  • Hides internal implementation, shows only essential features
  • Achieved via abstract classes and interfaces
    • Example: Pressing car brake ‚Äî you don‚Äôt know how it works internally
interface Car {
    void applyBrake();
    void incSpeed();
}
class CarImpl implements Car {
    public void applyBrake() { ... }  // internal steps hidden
}

üî∑ 2nd Pillar: Encapsulation

  • Binding data and methods into a single unit (class)
  • Achieved by:
    • Keeping variables private
    • Providing public getters and setters
class Dog {
    private String colour;

    public String getColour() { return this.colour; }
    public void setColour(String colour) { this.colour = colour; }
}
Dog lab = new Dog();
lab.setColour("black");

üî∑ 3rd Pillar: Inheritance

  • Child class inherits properties and methods from parent class
  • Promotes code reusability
  • Use extends keyword
class Vehicle {
    boolean engine;
    boolean getEngine() { return engine; }
}
class Car extends Vehicle {
    String type;
    String getCarType() { return type; }
}
Car swift = new Car();
swift.getEngine(); // works
  • Types:
    • Single
    • Multilevel
    • Hierarchical
    • Multiple (via interfaces only in Java)

üî∑ 4th Pillar: Polymorphism

  • Poly = many, morph = forms
  • Method Overloading (Compile Time):
    • Same method name, different parameters
  • Method Overriding (Runtime):
    • Same method, same parameters in subclass
// Overloading
int doSum(int a, int b) { return a + b; }
int doSum(int a, int b, int c) { return a + b + c; }

// Overriding
class A {
    int getEngine() { return 1; }
}
class B extends A {
    int getEngine() { return 2; } // overrides
}
B obj = new B();
obj.getEngine(); // returns 2

üî∑ Relationships

  • Is-A ‚Üí via Inheritance (e.g., Dog is-an Animal)
  • Has-A ‚Üí via Composition/Aggregation (e.g., Car has-an Engine)
    • Association: General relation
    • Aggregation: Objects can survive independently
    • Composition: One object‚Äôs lifecycle depends on the other

‚úÖ OOPS Concepts in Java (Detailed Notes)

üîπ 1. Class and Object

  • A Class is a blueprint or template for creating objects.
  • An Object is an instance of a class.
  • Object = State (fields) + Behavior (methods)

üß† Interview Tip: Be ready to write custom classes like Employee, Student, etc., and create objects from them.

    String brand;
    int speed;

        speed += 10;

üîπ 2. Encapsulation (Data Hiding)

  • Wrapping data (variables) and code (methods) together inside a class.
  • Achieved by:
    1. Making variables private
    1. Providing public getters and setters

üß† Why?

  • Protects data from unintended modification.
  • Improves maintainability and security.
class BankAccount {
    private double balance;

    public double getBalance() { return balance; }
    public void deposit(double amount) {
        if (amount > 0) balance += amount;
    }
}

üß† Interview Ask: Explain how encapsulation is used in real-world applications (e.g., bank account security).

üîπ 3. Abstraction (Hiding Implementation)

  • Shows only the relevant details to the user.
  • Hides internal implementation.
  • Achieved using:
    • Abstract Classes
    • Interfaces

üß† Why?

  • Focuses on what an object does rather than how it does it.
interface Vehicle {
    void start();
    void stop();
}
class Bike implements Vehicle {
    public void start() { System.out.println("Bike started"); }
    public void stop() { System.out.println("Bike stopped"); }
}

üß† Interview Tip: Be prepared to implement a real-life interface like PaymentMethod, MediaPlayer, etc.

üîπ 4. Inheritance (Code Reuse)

  • One class (child) can inherit fields and methods from another class (parent).
  • Use extends keyword.
  • Promotes code reusability.

üìö Types of Inheritance:

  • Single: One child - one parent.
  • Multilevel: Child of child class.
  • Hierarchical: Multiple children from one parent.
  • ‚ùå Multiple (Java doesn‚Äôt support directly; use interfaces).
class Animal {
    void eat() { System.out.println("Eating..."); }
}
class Dog extends Animal {
    void bark() { System.out.println("Barking..."); }
}

üß† Interview Ask: Can constructors be inherited? No. Constructors are not inherited but super() can call parent constructor.

üîπ 5. Polymorphism (Many Forms)

  • Same method behaves differently based on context.

üëâ Types:

  1. Compile-Time (Static) ‚Üí Method Overloading
  1. Runtime (Dynamic) ‚Üí Method Overriding

Method Overloading (same method, diff params):

class Printer {
    void print(String s) { System.out.println(s); }
    void print(int i) { System.out.println(i); }
}

Method Overriding (same method in parent-child):

class Animal {
    void sound() { System.out.println("Some sound"); }
}
class Cat extends Animal {
    void sound() { System.out.println("Meow"); }
}

üß† Interview Ask: Explain overriding rules: same method signature, runtime decision, @Override annotation, etc.

üîπ 6. Relationships: "IS-A" vs "HAS-A"

‚úÖ IS-A

  • Inheritance-based
  • Dog IS-A Animal

‚úÖ HAS-A

  • Composition/Aggregation
  • Car HAS-A Engine

üß† Example:

class Engine { }
class Car {
    private Engine engine = new Engine(); // Has-a
}
  • Association: General connection
  • Aggregation: Weak ownership (can exist independently)
  • Composition: Strong ownership (lifespan tied)
