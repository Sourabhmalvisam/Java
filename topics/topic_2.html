How Java Program Works and its 3 Important Components (JVM, JRE and JDK)

✅ JAVA Platform Basics: JDK, JRE, JVM

These are the three pillars of the Java platform. Every Java developer must understand them clearly—this is often asked in interviews.

🔷 1. JVM (Java Virtual Machine)

🔹 What is it?

  • It is an abstract machine (doesn't physically exist).
  • Responsible for executing Java bytecode on any platform.
  • Converts .class (bytecode) into machine-specific code.

🔹 Key Features:

  • Platform dependent (Mac JVM ≠ Windows JVM)
  • Bytecode is platform-independent, but JVM is not.
  • JVM Includes JIT Compiler:
    • JIT = Just-In-Time Compiler
    • Optimizes bytecode to machine code at runtime.

🔹 Diagram:

Java Code (.java) --> Compiler --> Bytecode (.class) --> JVM --> Machine Code --> Output

🔷 2. JRE (Java Runtime Environment)

🔹 What is it?

  • Provides the runtime environment to run Java programs.
  • Contains:
    • JVM
    • Core libraries (Java APIs like java.lang, java.util)

🔹 Limitation:

  • JRE allows you to run Java programs but you cannot develop or compile code with it.

🔹 Usage:

  • End-users who want to run Java apps need only JRE.

🔷 3. JDK (Java Development Kit)

🔹 What is it?

  • Complete development kit for Java developers.
  • Includes:
    • JRE (JVM + libraries)
    • Compiler (javac)
    • Debugger, tools, documentation

🔹 Diagram:

JDK = JRE (JVM + Libraries) + Compiler + Development Tools

🔹 Usage:

  • Required for developing, compiling, debugging Java programs.

🔷 Relationship Summary

ComponentIncludesPurpose
JVMCore runtime engineRuns bytecode
JREJVM + LibrariesRuns Java programs
JDKJRE + Dev ToolsDevelops and runs Java programs

🔷 JDK Variants (As per PDF)

  • JSE (Java Standard Edition):
    • Core Java: OOPs, collections, multithreading, etc.
  • JEE (Java Enterprise Edition):
    • Web applications: Servlets, JSP, EJB, APIs
  • JME (Java Micro Edition):
    • Mobile and embedded devices

🧠 Interview Tip: Be ready to explain what happens behind the scenes when you run a Java program:

Compiler → Bytecode → JVM → Machine Code
