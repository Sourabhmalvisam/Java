ðŸ”¹ What is a Method?

  • A block of code that performs a specific task.
  • Promotes reusability, modularity, and readability.

ðŸ”¹ Syntax

accessSpecifier returnType methodName(type param1, type param2) {
    // method body
    return result;
}

ðŸ”· Method Components

ComponentDescription
Access Specifierpublic, private, protected, default
Return Typeint, String, void, etc.
Method Nameshould be a verb, use camelCase
Parametersoptional list of input arguments
Bodyblock of code to execute

ðŸ”· Types of Methods

  1. System-defined (e.g. Math.sqrt())
  1. User-defined (custom logic)
  1. Overloaded Methods
    • Same name, different parameter list
    • Return type does not matter
void print(String s) {}
void print(int i) {} // Valid overloading
  1. Overridden Methods
    • Redefining parent class method in child class
    • Used in runtime polymorphism
    • Must match signature
  1. Static Methods
    • Belong to the class, not object
    • Can be accessed without object
    • Cannot access non-static members
    • Cannot be overridden
  1. Final Methods
    • Cannot be overridden by child classes
    • Used to lock implementation
  1. Abstract Methods
    • No body
    • Must be inside an abstract class
    • Implemented by subclass
  1. Varargs (Variable Arguments)
    • Accepts zero or more arguments
    • Must be the last parameter in method
void sum(int... numbers) { }

âœ… CONSTRUCTORS in Java

ðŸ”¹ What is a Constructor?

  • Special method that initializes an object.
  • Called when you use new keyword.
  • No return type, and name must match the class name.
class Car {
    Car() { System.out.println("Constructor called"); }
}

ðŸ”· Key Rules

  • Cannot be static, final, or abstract
  • Constructors cannot be inherited
  • Constructors can be overloaded

ðŸ”· Types of Constructors

  1. Default Constructor
    • Provided by Java only if no other constructor is defined
    • No parameters
  1. No-arg Constructor
    • Explicitly written constructor with no parameters
  1. Parameterized Constructor
    • Takes arguments to set initial values
  1. Overloaded Constructors
    • Multiple constructors with different parameter lists
  1. Private Constructor
    • Prevents object creation outside the class
    • Used in Singleton Design Pattern
private ConstructorExample() {}

public static ConstructorExample getInstance() {
    return new ConstructorExample();
}

âœ… Using this()

  • Calls another constructor in same class

    Student() { this("Ganesh"); }

âœ… Using super()

  • Calls parent class constructor (must be first line)
  • If not written explicitly, Java calls super() by default
